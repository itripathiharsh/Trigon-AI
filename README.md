# AI Operations Assistant (GenAI Multi-Agent System)

An intelligent, agentic system designed to handle complex user queries by orchestrating specialized tools. Built with a robust **Planner-Executor-Verifier** architecture, this assistant integrates real-time data from GitHub, Weather, News, and Media APIs to provide verified, high-quality responses.

* **Python 3.9+** installed.

## üèóÔ∏è Architecture Explanation

This system utilizes an **Agentic Workflow** to ensure accuracy and prevent LLM hallucinations.

### 1. Planner Agent (The Brain)

* **Role**: Analyzes the user's natural language input.
* **Logic**: Breaks the request into a structured JSON execution plan (step-by-step tool calls).
* **LLM**: Powered by **Llama-3.3-70b (via Groq)** for high-reasoning planning.

### 2. Executor Agent (The Hands)

* **Role**: Executes the plan generated by the Planner.
* **Logic**: Routes tasks to the specific third-party APIs. It handles argument extraction and collects raw data from all integrated tools.

### 3. Verifier Agent (The Judge)

* **Role**: Audits the data returned by the Executor.
* **Logic**:
* **Success Path**: If data is valid, it formats it into a professional Markdown report.
* **Self-Correction Path**: If the tool data is empty or irrelevant, it generates a "Retry Instruction" and sends it back to the Planner to try a different approach.

## ‚ö†Ô∏è Known Limitations & Tradeoffs

* **Rate Limits**: Since we are using free-tier API keys (Groq/Currents), sending 5+ rapid requests might trigger a 429 Rate Limit error.
* **Foursquare Deprecation**: The Foursquare Places API was deprecated in favor of a more stable manual data aggregation to ensure 100% uptime for the assignment demo.
* **Context Window**: To maintain speed, the Verifier only processes the top 5 news articles or search results at a time.
